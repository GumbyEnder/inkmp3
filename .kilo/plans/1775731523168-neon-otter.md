# Plan: Fix 'f' key to toggle favorite on frontpage

## Problem

When on the frontpage (HomeLayout) and pressing 'f', the favorite toggle doesn't work. The keybinding handler for TOGGLE_FAVORITE is only defined in PlayerControls.tsx, not in HomeLayout.tsx.

## Solution

Add a keybinding handler in HomeLayout.tsx for the 'f' key (TOGGLE_FAVORITE) that toggles the currently playing track as a favorite.

## Changes Needed

### File: `source/components/layouts/HomeLayout.tsx`

1. **Add `toggleFavorite` to destructured `useFavorites()`** (line 21):

   ```typescript
   // Current:
   const {favorites} = useFavorites();

   // Change to:
   const {favorites, toggleFavorite} = useFavorites();
   ```

2. **Add keybinding for TOGGLE_FAVORITE** (around line 112, after SEARCH handler):

   ```typescript
   useKeyBinding(KEYBINDINGS.TOGGLE_FAVORITE, () => {
   	if (playerState.currentTrack) {
   		toggleFavorite(playerState.currentTrack);
   	}
   });
   ```

3. **Optional: Fix footer hint** (line 314) - currently says "Shift+F: Favorites" but should say "f: Favorites" since the key is just 'f', not Shift+F.

## Verification

After the fix, pressing 'f' while on the frontpage with a track playing will toggle that track as a favorite.
